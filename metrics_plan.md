# DeskDrop — Metrics Plan (Lean Startup + Google Analytics)

This document outlines how to measure the quality, growth, and product-market fit of DeskDrop using the **Lean Startup** methodology. We focus on **Actionable Metrics** (which help make decisions) rather than **Vanity Metrics** (which only look good on paper).

---

## 1. Raw Metrics (The Data to Collect)

These are the discrete events and properties that should be logged as they happen.

| Category | Raw Metric | Description |
| :--- | :--- | :--- |
| **Acquisition** | `user_session_start` | Triggered when a unique visitor lands on the site. |
| **Acquisition** | `auth_signup_success` | Triggered when a new user successfully creates an account. |
| **Activation** | `space_detail_view` | Triggered when a user views a specific space (ID). |
| **Activation** | `filter_applied` | Triggered when a user uses the capacity or price filters. |
| **Retention** | `cart_item_added` | Triggered when "Add to Cart" is clicked. |
| **Retention** | `cart_item_removed` | Triggered when an item is deleted from the cart. |
| **Revenue** | `booking_attempted` | Triggered when "Confirm Booking" is clicked. |
| **Revenue** | `booking_confirmed` | Triggered when the DB creates a confirmed booking record. |
| **System** | `booking_error_clash` | Triggered when a booking fails due to a date/time conflict. |

## 2. Actionable Metrics (The Insights)

These calculated metrics tell us if the product is improving or where user friction exists.

### A. Activation Rate
*   **Definition**: The percentage of signed-up users who proceed to view at least one space detail page.
*   **Formula**: `(Unique Users with space_detail_view) / (Unique Users with auth_signup_success)`
*   **Why it matters**: If this is low, your onboarding or landing page isn't convincing users that the spaces are worth looking at.

### B. Cart-to-Booking Conversion (Checkout Efficiency)
*   **Definition**: The percentage of users who add an item to their cart and successfully complete a booking.
*   **Formula**: `(Total booking_confirmed) / (Total cart_item_added)`
*   **Why it matters**: High abandonment here indicates friction in the checkout process (e.g., hidden costs, complex forms, or confusion).

### C. Booking Conflict Rate (Quality Metric)
*   **Definition**: The frequency of failed bookings due to scheduling overlaps.
*   **Formula**: `(Total booking_error_clash) / (Total booking_attempted)`
*   **Why it matters**: If this is high, the "Available" status on the landing page/detail page is stale, leading to a frustrating user experience. It signals a need for better real-time availability UI.

### D. Revenue per Active User (RPau)
*   **Definition**: The average value extracted from users who are actively browsing.
*   **Formula**: `(Total GMV from bookings) / (Unique Users with space_detail_view)`
*   **Why it matters**: Measures how effective the "inventory" (spaces) is at generating revenue from the traffic you have.

### E. Viral Coefficient (If Referral implemented)
*   **Definition**: The number of new users generated by existing users.
*   **Formula**: `(Total referral_signups) / (Total active_users)`
*   **Why it matters**: Tells you if the app can grow organically (Lean Startup focus on "Engines of Growth").

## 3. The Build-Measure-Learn Loop

Following this plan allows you to iterate effectively:

1.  **Hypothesis**: "Reducing the number of steps in the checkout layout will increase our **Checkout Efficiency**."
2.  **Experiment**: Implement a one-page checkout (`/checkout` refactor).
3.  **Measure**: Compare the **Cart-to-Booking Conversion** before and after the change.
4.  **Learn**: If the metric improves by >5%, pivot to optimizing the next friction point (e.g., Filtering). If it stays the same, investigate why users are still dropping off.

## 4. Google Analytics 4 (GA4) Integration

Using Google Analytics alongside Supabase allows for tracking the "Upper Funnel" (anonymous browsing behavior) before users sign up or book.

### Setup (Next.js 14)

1.  **Dependency**: Install `@next/third-parties`.
    ```bash
    npm install @next/third-parties@latest
    ```
2.  **Root Layout**: Add the `GoogleAnalytics` component in `src/app/layout.tsx`.
    ```tsx
    import { GoogleAnalytics } from '@next/third-parties/google'

    export default function RootLayout({ children }) {
      return (
        <html lang="en">
          <body>{children}</body>
          <GoogleAnalytics gaId="G-XXXXXXXXXX" />
        </html>
      )
    }
    ```

### Event Mapping Table

| Metric | GA4 Recommended Event | Custom Dimensions / Parameters |
| :--- | :--- | :--- |
| **Signup** | `sign_up` | `method: "email"` |
| **Space View** | `view_item` | `item_id`, `item_name`, `price` |
| **Add to Cart** | `add_to_cart` | `item_id`, `item_name` |
| **Checkout Start** | `begin_checkout` | `value`, `currency` |
| **Booking Confirmed** | `purchase` | `transaction_id`, `value`, `items` |
| **Booking Error** | `exception` | `description: "booking_clash"` |

## 5. Implementation Notes (Supabase & GA4)

To collect these, you can:
- Use **Next.js Middleware** or **React useEffect** to log events.
- Create an `app_events` table in Supabase to store these logs.
- Use **Supabase Real-time** or simple aggregation queries (`COUNT`, `SUM`) to generate these metrics in a dashboard.
